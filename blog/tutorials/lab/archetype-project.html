  • Generate the sample archetype project
  • Build/Deploy the archetype project to create first headless adaptive form

Generate the sample archetype project

  • Open the command prompt and run the below command to create an Experience Manager Forms as a Cloud Service project. Use archetypeVersion 37 or later
mvn -B org.apache.maven.plugins:maven-archetype-plugin:3.2.1:generate \
-D archetypeGroupId=com.adobe.aem \
-D archetypeArtifactId=aem-project-archetype \
-D archetypeVersion=37 \
-D appTitle=myheadlessform \
-D appId=myheadlessform \
-D groupId=com.myheadlessform \
-D includeFormsheadless="y"
  • The archetype project comes with the auto-public repository configured. To setup the repository in your Maven settings, refer to official docs

Build/Deploy the archetype project to create first headless adaptive form

  • Open Command line and navigate to the directory of your headlessform project created in the archetype
  • Building the project will take some time since it requires downloading multiple dependencies from different repositories.
mvn clean install
  • You can optionally deploy the project beforehand to your local aem instance to save time during the lab. To deploy to your local development environment, use the following command:
mvn -PautoInstallPackage clean install
