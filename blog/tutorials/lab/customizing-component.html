Front-end module customisation

  • The :type field above represents custom component. Let's change the [Archetype project]\ui.frontend.react.forms.af module to include this new component

  • Let's use the date picker component from material UI. To use this component, run the following command inside [Archetype project]/ui.frontend.react.forms.af module,

npm install @date-io/date-fns@latest \
@date-io/moment@1.1.0 @material-ui/core@latest \
date-fns@next material-ui-pickers@latest \
moment@2.24.0 --force

  • Create a folder called material-ui inside src. Add files Date.tsx and mappings.ts. Date.tsx would contain code to render date component with material ui and mappings.ts contains mappings to map the :type property value with the react component class

  • Let's update the adaptive form renderer to use these mappings. Navigate to [Archetype project]/ui.frontend.react.forms.af/src/Form.tsx and import the mappings object (ie) replace the following line,

import {mappings} from "@aemforms/af-react-components";

with

import mappings from "./material-ui/mappings";

  • To see these changes in action on local development server, let's first upload the form model definition with the new changes in AEM as a dam asset. After you are logged in, in the upper-left corner, tap Adobe Experience Manager > Assets > Files. Tap Create and select Files. Upload the file original.form.json as DAM asset

  • After the upload is complete. Navigate to Adobe Experience Manager > Forms > Forms & Documents and open the contact form in editor. Tap the Adaptive Forms Container component and Tap Properties. In the properties explorer, select the original.form.json dam asset path for the Forms Runtime Document Path option. The container component displays a rendition of the form

  • Before deploying the changes to AEM, let's validate the customisation changes on local development server by navigating to the ui.frontend.react.forms.af project and running the following command:
npm run start

  • Once the above command is run, wait for the following message webpack compiled with 4 warnings. This means that the local development server has started successfully.

  • Rename the original.form.json file back to original in the archetype project to avoid filevault-package-maven-plugin errors.

  • Let's deploy the changes to AEM using the following command

mvn -PautoInstallPackage clean install
